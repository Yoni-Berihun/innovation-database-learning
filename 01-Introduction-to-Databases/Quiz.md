## ✅ Answer Key

&lt;details&gt;

### Question 1
**Correct answer:** An organized collection of data that can be easily accessed, managed, and updated

**Why:** This is the core definition. A database isn't random files, isn't a spreadsheet, and isn't a programming language. It's a structured system for storing and working with data.

---

### Question 2
**Correct answer:** Database Management System

**Why:** DBMS stands for Database Management System — the software that creates, manages, and protects databases. PostgreSQL is an example of a DBMS.

---

### Question 3
**Correct answer:** PostgreSQL

**Why:** PostgreSQL is a Relational Database Management System (RDBMS). Word is a word processor, Chrome is a web browser, and Photoshop is an image editor. None of them manage databases.

---

### Question 4
**Correct answer:** Databases can handle more complex relationships, more users, and larger amounts of data safely

**Why:** Spreadsheets are great for simple, small datasets managed by one person. Databases are built for scale, relationships, multiple users, security, and reliability. Both can be free or paid.

---

### Question 5
**Correct answer:** The list of all enrolled students and their information

**Why:** A database stores structured, searchable data. A logo, color scheme, or font are files/assets — not structured data that needs to be queried, related, or validated.

---

### Question 6
**Correct answer:** Relational database

**Why:** Relational databases (like PostgreSQL, MySQL, SQL Server) use tables with rows and columns. They are specifically designed to handle relationships between tables using keys.

---

### Question 7
**Correct answers:**
- Relational databases enforce rules that prevent invalid data
- Relational databases allow many users to access data safely at the same time
- Relational databases can recover from crashes and hardware failures

**Why:** These are three core advantages of relational databases. The fourth option is false — text files are NOT always faster; for complex queries and large datasets, databases are dramatically faster.
